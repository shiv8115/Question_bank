## 🧑‍🏫 How to Use This Question Bank During Interviews

### ✅ Before the Interview
1. **Select 8–12 questions per section** based on the role level (junior/senior).
2. **Mix types**: Start with 2–3 basic questions to warm up, then move to intermediate, then 1–2 practicals.
3. **Avoid using all questions from one difficulty level** — variety keeps the interview dynamic.
4. **For seniors**, include system design and coding standards sections.
5. **Print or open** the relevant section files; avoid scrolling during the interview.

### ✅ During the Interview
- For each question, **give the candidate 30–60 seconds** to think before answering.
- Use **follow-up probes**: "Can you explain why?", "What would happen if…?", "Have you used this in production?"
- For coding questions, let the candidate **write pseudocode first**, then actual code.
- **Rate answers** on a simple 1–3 scale: 1 = Missed, 2 = Partial, 3 = Clear

### ✅ Across 10 Back-to-Back Candidates
- **Rotate questions** between candidates: pick different 8–12 from the pool each time.
- Use a **score sheet**: write candidate name, question ID, and score.
- Mark questions used ✓ to track which ones you've asked today.
- Reserve the **debugging and scenario questions** for mid-interview when the candidate is warmed up.
